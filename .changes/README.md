# Changelog Fragments

Add one short Markdown fragment per production-meaningful change. Fragment names should use the
date and a short slug, for example `2026-06-25-foundation-setup.md`.

Use this shape:

```markdown
# <Short Change Title>

Date: YYYY-MM-DD
Type: docs | feature | fix | ops | security | chore

- What changed.
- Any verification that matters.
```

Fragments are release source material. Do not add automation until release cadence needs it.
